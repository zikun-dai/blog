## OpenAI

### ChatGPT 网页端

> [!WARNING]
> 代码相关的问题（pylance 标红、运行报错、虚拟环境、测试脚本等），通过网页端 + 手动执行 + 反馈给 AI 的流程太慢。
> 建议直接在 Codex 里让 AI 自己执行命令；如需联网或多工具协作，可考虑 multi-agent。
>
> 委派给 Codex 的 prompt：
>
> ```
> 给出基本的背景信息（创建虚拟环境的命令，解决报错后的策略，重新创建还是直接 update）。
> 如果遇到报错，你自己读取报错信息并进行解决。
> 解决问题之后继续安装环境，直到环境彻底安装成功再结束。（最后再指定这个自迭代）
> ```

:::tip
一般使用 `thinking + 网页搜索` 即可，pro 思考太慢了。
:::

- 5.1pro：经常过度思考，耗时长且次数限制。
- 5.2pro：还没怎么用过。
- 5.2 thinking：建议勾选“网页搜索”，回答更充分；否则可能不思考就快速回答。

### Codex

:::tip
一般选择代理模式，5.2-codex，推理强度选中（非常简单的修改选低）。
不希望大改框架时，可在对话框指定“做最小粒度的修改”。
:::

- insert TODO：非常好用，适合精确定位编辑位置。
- skills：还没实际用过，听说很夯。
- 云端：很好用，但日常用不到；当前已知问题：
  - 本地领先于云端时，基于本地修改创建云端任务会失败。
  - 在本地 Codex 上下文直接切换云端任务会失败，需要新开对话框选择云端。
  - 云端修改完成后，本地继续让云端修改会报错，只能由本地继续。
  
  > [!TIP]
  >
  > 不过本地可以直接查看云端进度、完成后也能直接应用云端修改到本地，无需GitHub PR（手机远程时可以选择创建PR新分支）；
  > 		在codex网页端似乎是可以让云端接着改的；
  
  

## Gemini

生成软件前，先简单描述需求，让它生成功能需求说明书（PRD），逐步完善想法后再勾选 Canvas 生成页面。

:::tip
勾选 Canvas 时最好写明“使用 Canvas 生成网页”。否则可能在正文里插入代码，而不是在 Canvas 预览。
:::
